# PRD.md — 반도체 시료 생산주문관리 시스템

## 1. 배경

가상의 반도체 회사 **S-Semi**는 다양한 종류의 반도체 시료(Sample)를 생산하여 연구소,
팹리스(Fabless) 업체, 대학 연구실 등의 고객에게 납품한다. 시료는 주문이 들어오면 웨이퍼
공정 설비를 통해 제작되고, 검수를 거쳐 고객에게 출고된다.

최근 주문량이 급증하면서 엑셀과 메모장으로 주문을 관리하던 기존 방식으로는 다음과 같은
문제가 발생했다:

- 주문 처리 여부를 파악하기 어렵다.
- 공정 예약 후 완료 시점을 알 수 없다.
- 재고가 충분한데도 불필요한 추가 공정이 진행된다.

이를 해결하기 위해 더 체계적인 시료 관리를 위한 **"반도체 시료 생산주문관리 시스템"**을
개발한다.

## 2. 역할별 흐름

```
고객 --(시료 요청, 이메일)--> 주문 담당자 --(주문서 전달)--> 생산 담당자
                                    ^                              |
                                    └────────(승인/거절)───────────┘
```

- **고객**: 필요한 시료를 요청하는 시료 요청자.
- **주문 담당자**: 요청에 맞게 주문서를 작성·관리.
- **생산 담당자**: 개발 시료를 등록하고, 주문 수신 후 승인 또는 거절.

## 3. 시스템 개요

- 콘솔(CLI) 기반으로 동작하며, 담당자가 직접 명령을 입력해 시료를 등록하고 주문을 처리한다.
- 생산 라인은 하나이며, 시료를 하나씩 순차 생산한다 (단일 라인).
- 주문이 들어온 시료에 대해서만 생산한다 (재고 없는 선제 생산 없음).

## 4. 주문 상태 흐름

```
[주문 등록] --> RESERVED
                  |
              승인 여부
              /       \
          거절          승인
           |              |
        REJECTED      재고 확인
                       /        \
                  재고 충분   재고 부족
                     |            |
                 CONFIRMED   PRODUCING
                     |            |
                 출고 준비    생산 완료(생산 요청)
                     |            |
                 출고 처리 <------┘
                     |
                  RELEASE (고객 도달)
```

| 상태 | 의미 |
|------|------|
| `RESERVED` | 주문 접수 |
| `REJECTED` | 주문 거절 — 정상 흐름 외 상태이며 모니터링 대상에서 제외 |
| `PRODUCING` | 주문 승인 완료 + 재고 부족으로 생산 중 |
| `CONFIRMED` | 주문 승인 완료 + 출고 대기 중 (재고 충분 즉시 전환 또는 생산 완료 후 전환) |
| `RELEASE` | 출고 완료 |

## 5. 메인 메뉴

시스템 진입 시 전체 시료에 대한 요약 정보(등록 시료 수, 총 재고, 전체 주문 수, 생산라인
대기 건수 등)를 표시하고, 아래 메뉴로 진입한다. 화면 구성은 자유롭게 결정한다.

| 메뉴 | 설명 |
|------|------|
| 시료 관리 | 새로운 시료 등록, 목록 조회, 이름 검색 |
| 시료 주문 | 고객 주문 접수 (`RESERVED` 생성) |
| 주문 승인/거절 | 생산 라인 담당자의 승인·거절 처리 |
| 모니터링 | 상태별 주문 수 및 시료별 재고 현황 확인 |
| 생산 라인 | 현재 생산 중인 시료 및 대기 중인 생산 큐 확인 |
| 출고 처리 | `CONFIRMED` 상태 주문에 대해 출고 실행 |

## 6. 기능 상세

### 6.1 시료 관리

시료(Sample)는 이 시스템의 가장 기본이 되는 단위다. 각 시료는 고유한 이름과 속성을 가지며,
시스템에 등록된 시료만 주문 가능하다.

- **시료 등록**: 새로운 시료를 시스템에 추가. 속성값 — 시료 ID, 이름, 평균 생산시간, 수율.
- **시료 조회**: 등록된 모든 시료 목록과 현재 재고 수량을 함께 표시.
- **시료 검색**: 이름 등 속성으로 특정 시료를 검색.

> **수율** = 정상적인 시료 수 / 총 생산 시료 수. (예: 100개 생산 중 정상 90개 → 0.9)

### 6.2 시료 주문

고객이 시료를 요청하면 주문 담당자가 주문을 생성한다.

- **입력값**: 시료 ID, 고객명, 주문 수량.
- 생성 직후 주문 상태는 `RESERVED`.

### 6.3 주문 승인/거절

- **접수된 주문 목록**: `RESERVED` 상태의 주문 목록을 표시.
- **주문 승인**: 특정 주문을 승인하면 재고 상황에 따라 자동 분기한다.
  - 재고 충분 → 즉시 `CONFIRMED`.
  - 재고 부족 → 생산 라인에 자동 등록, `PRODUCING`으로 전환.
- **주문 거절**: 특정 주문을 거절하면 즉시 `REJECTED`로 전환.

### 6.4 모니터링

담당자가 현재 시스템 상태를 한눈에 파악할 수 있도록 구성한다.

- **주문량 확인**: 상태별(`RESERVED`/`CONFIRMED`/`PRODUCING`/`RELEASE`) 목록 확인.
  `REJECTED`는 유효한 주문이 아니므로 무시.
- **재고량 확인**: 시료별 현재 재고 수량과, 주문 대비 재고 수준에 따른 상태 표기.
  - **여유**: 주문 대비 재고 충분.
  - **부족**: 주문 대비 재고 수량 부족.
  - **고갈**: 재고 수량이 0.

### 6.5 생산 라인

생산 라인 정보를 표시한다. 주문량에 대한 부족분을 생산하되, 수율 및 오차를 고려하여 시료를
생산한다.

- **실 생산량** = `ceil(부족분 / 수율)`
- **총 생산 시간** = 평균 생산시간 × 실 생산량
- 생산 완료 시 주문 상태를 `PRODUCING → CONFIRMED`로 변경.
- **생산 현황 표기**: 현재 생산 중인 시료 정보(예: 주문 정보, 현재까지의 생산량 등). 표기
  수준은 자율적으로 결정.
- **대기 주문 확인**: 생산 큐를 이용해 대기 중인 목록을 출력.
  - **스케줄링 전략**: FIFO(선입선출).

### 6.6 출고 처리

`CONFIRMED` 상태(재고가 이미 확보된) 주문에 대해 출고를 처리한다.

- 특정 주문에 대해 출고를 실행하면 주문 상태가 `RELEASE`로 전환된다.
